What you need:
 - linux server
 - java jdk version 1.6
 - ant version 1.8
 - scala version 2.10 - http://www.scala-lang.org/
 - ninthdrug.jar - https://github.com/ninthdrug/ninthdrug

Optional:
 - postgresql
 - sudo rights to postgres user

Configuring:
   The installation adds the path to the main doom configuration to your $HOME/.ninthdrug file.
   That should be ${install.dir}/conf/taakapp.conf.  Edit this as you see fit.

   The doom database can be created by running the create_fido_db.sh script.  
   You will need sudo rights to the postgres user for this step.
   - create_taakapp_db.sh

Edit the conf/taakapp.conf file to set the correct user and password for the taak database.
Edit appdevkit/daogenerator.conf 
Edit appdevkit/sql/create_tables.sh

Is written for Postgres.
- Create a user and a database for that user
- Create database and tables

Code Generator:
 
- Use this command to generate a db.description file
     psql taak taak -h localhost < describe_tables.sql > db.description
- Edit the daogenerator.conf
- Run the daogenerator.sh <tablename> for each table
- Publish code to src directory with publishdaocode.sh

Building from source:
 - Extract the tar ball
 - Change to the taakapp/trunk directory
 - Edit the conf/taakapp.conf file to fit your installation
 - Invoke ant
 - Invoke ant deploy


Defaults:

taakapp database
  -tasks table
  -users table

  each table should have
  -id Int
  -active Boolean
